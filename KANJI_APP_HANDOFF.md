# 中3漢字書き取りアプリ 引き継ぎ仕様書

最新版: **v1.7**  
更新日: **2026-07-20**  
公開先: `https://biscobeil6.github.io/kanji20260718/`

## 1. アプリ概要

中学3年生向けの漢字書き取りWebアプリ。iPadを主用途とし、iPhoneでも使用する。Apple Pencil・指・一般的なタッチペン・マウスに対応する静的Webアプリで、GitHub Pagesから公開する。

主な機能:
- 手書きキャンバス
- 解答表示後も自分の筆跡を保持
- 一つ戻す / 全消去
- 自己採点 ○△×
- 周回出題
- ランダム出題
- 苦手一覧
- 10問ごとのごほうび画像
- 端末内保存

## 3. 現在の出題仕様

### 周回出題
- 1周の中で同じ問題を重複させない。
- 全240問を出題し終えると、成績を消さず自動で次の周へ進む。
- 周回数と周回内の進捗を端末に保存する。
- ○△×は成績へ反映する。

### ランダム出題
- 全240問からランダムに出題する。
- 同じ問題が連続・再登場する場合がある。
- ○△×は成績へ反映する。

### 苦手一覧
- **直近の自己採点が△または×の問題だけ**表示する。
- 次回その問題に○を付けると、一覧から自動で外れる。
- 累計×回数、正答率、復習期限などの複雑な判定は使用しない。
- 独立した「復習」機能・ボタンは作らない。

## 4. 問題データ

- 総数: 240問
- 通常書き取り: 150問
- 同音異義語: 30問
- 四字熟語: 60問
- 問題データは `questions.json` に分離。
- 問題追加・修正は原則 `questions.json` のみ変更する。

## 5. 手書き入力仕様

- Apple Pencil・指・マウスに対応。
- Apple PencilはPointer EventsとTouch Eventsの両経路で受ける。
- 指とApple Pencilを可能な範囲で識別する。
- 筆圧は線幅に使わない。
- 線幅は固定。
- 短い接触・点も描画する。
- Apple Pencilの一画取りこぼし対策を実装済み。
- 描画中は増分描画し、毎回の全再描画は避ける。
- 一つ戻す・全消去時のみ再描画する。

### Apple Pencil関連で壊しやすい箇所
`index.html` の入力イベント処理を変更する時は、以下を維持する。
- Pointer EventsとTouch Eventsの二重受信
- 二重描画防止
- 固定線幅
- pointercancel / touchcancel / visibilitychange 後の状態解除
- 新しい入力開始時に古い入力状態を引きずらない処理

## 6. 成績・保存

- 保存先: ブラウザの `localStorage`
- 主キー: `kanjiWritingApp.v1`
- iPhoneとiPadは別保存で、同期しない。
- GitHub上のファイル更新では通常消えない。
- URLの `?v=9` などを変えても通常消えない。
- Safariのサイトデータ削除、またはアプリ内「成績リセット」で消える。
- 成績リセット時は、○△×・苦手一覧・周回進捗等を消す。
- 問題データ・画像ファイルそのものは消えない。

## 7. ごほうび画像

- 10問達成ごとに表示。
- 現在は10枚。
- `images/reward01.jpeg` ～ `reward10.jpeg`。
- `reward02.jpeg` は2026-07-20に最後に差し替えた画像。
- 画像追加・削除時は `images/` と `rewards.json` を同時に更新する。

## 8. GitHub Pages更新手順

### アプリ本体を変える場合
1. 修正版を手元で `index.html` という名前にする。
2. GitHubの既存 `index.html` を上書きする。
3. `index.html.html` を作らない。
4. 公開後、キャッシュ回避用に末尾を変えて開く。

例:
```text
https://biscobeil6.github.io/kanji20260718/?v=9
```

`v=9` の数字はアプリ内バージョンを自動選択するものではなく、ブラウザに別URLとして再取得させるための目印。更新ごとに `v=10`, `v=11` のように変えてよい。

### 画像を変える場合
1. GitHubの `images` フォルダを開く。
2. 同じ名前の画像をアップロードして置き換える。
3. 枚数・名前を変える場合は `rewards.json` も更新する。

### iPhone / iPadでimagesフォルダが消えた場合
GitHubは空フォルダを保存しない。まず新規ファイル作成でファイル名に半角で

```text
images/.keep
```

と入力してコミットすると `images` フォルダができる。その後、`images` を開き、画像を複数選択してアップロードする。

## 9. よくある事故

- `index.html.html` をアップロードしてしまう。
- 古い `index.html` が残り、新版が公開されない。
- `images` を消してしまう。
- 画像を差し替えたが `rewards.json` のファイル名と合っていない。
- 全角の `？` や `／` をURL・パスに使う。
- ZIPだけをGitHubへ置き、展開した中身をリポジトリ直下へ配置していない。
- Safariのキャッシュで古い画面を見ている。

## 10. 修正時の原則

- 既存のApple Pencil入力改善を壊さない。
- 既存の成績データをなるべく引き継ぐ。
- 保存キーを変更する場合は移行処理を入れる。
- UI文言変更だけで済む時はデータ構造を変えない。
- 問題修正は `questions.json`。
- ごほうび画像変更は `images/` と `rewards.json`。
- アプリ挙動変更は主に `index.html`。
- 修正後は差し替え用ファイルと、全体ZIPの両方を作る。

```text
修正後は、GitHub差し替え用ファイルと、更新済み全体ZIPをください。
```

## 12. このZIPの扱い

- ZIPは保存・引き継ぎ用。
- 別セッションへ添付して読ませられるが、環境によってZIP内部の自動検索が不十分な場合がある。
- 確実性を上げるため、プロジェクトにはZIPに加えて `KANJI_APP_HANDOFF.md` を単体でも置く。
- 実際の修正時は、ZIPを展開した最新版一式を基準にする。
